# 养老险GPT 下一步实施清单

## 近期目标

1. 把首页的 4 个 Agent 卡片接到 FastAPI `/api/agents/featured`。
2. 实现对话页：会话列表、消息流、AGUI SSE、工具事件展示。
3. 实现知识库页：资料库、文档上传、解析状态、检索测试。
4. 实现学习页：课程列表、MD/PPT 课时、学习进度、测验。
5. Playground 先做“即将上线”页，再补 Agent 草稿编辑：模型配置、能力集、Skills、系统提示词、调试对话。
6. 建立 PostgreSQL schema migration，先落用户、知识库、课程、Agent、Skill、运行日志六条主线。
7. 统一前端运行事件展示，后续兼容 AGUI custom event 和结构化 digest。

## 设计约束

- 保持 Alpha派式工作台密度：信息可扫读，少做营销式大 Hero。
- 平台名固定为“养老险GPT”。
- 业务语言贴近养老险培训、企业年金、合规销售、投教课件。
- 工具结果前端展示优先结构化，不依赖自然语言解析。
- 首页保持克制，只承担“入口”和“能力预告”，不堆学习、排行、市场等复杂模块。
